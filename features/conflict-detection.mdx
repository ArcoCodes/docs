## Avoid double-booking

When creating a new event, Kairos checks your calendar for time overlaps. If there's a conflict, you'll know immediately — before the event is created.

## How it works

When you ask Kairos to create an event (or when it offers to create one from an email), it scans your calendar for overlapping events. If a conflict is found, you'll see a notification with:

- The new event details
- The existing event it conflicts with
- Three options:

| Option | What happens |
|---|---|
| **Replace old** | The existing event is updated or removed, and the new one is created |
| **Keep both** | Both events stay on your calendar (in case you can manage both) |
| **Don't create** | The new event is canceled and your calendar stays unchanged |

## When conflicts are detected

Conflict detection happens automatically in these situations:

- You ask Kairos to create or move a calendar event
- Kairos offers to create an event from a schedule-related email
- Production milestones are being added during deal negotiation

## Works with your flow

Conflict detection is especially useful when managing multiple brand deals. If two shoots overlap, or a brand call conflicts with an existing commitment, you'll catch it before making promises you can't keep.
